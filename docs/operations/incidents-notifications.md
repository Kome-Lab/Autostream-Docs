# インシデントと通知

配信中の異常を見逃さないために、Observability と通知先を設定します。通知は多すぎても運用しづらいため、最初は重要なものから有効にします。

## 通知したいもの

- サービスが offline になった
- 配信開始や停止に失敗した
- 録画や upload に失敗した
- 音声が一定時間途切れた
- encoder の FPS や bitrate が下がった
- ディスク容量が不足しそう

## 設定手順

1. Observability を起動します。
2. Control Panel で Observability が online になっているか確認します。
3. 通知先を登録します。
4. テスト通知を送ります。
5. 閾値を確認します。
6. テスト配信で heartbeat と metric が届くことを確認します。

## 通知が多いとき

- 閾値を少しゆるめます。
- 重要度の低い通知を無効にします。
- 同じ原因の通知が重複していないか確認します。
- 復旧通知を使うか見直します。

## 対応記録

インシデント対応では、誰が何を確認したかを残すと再発時に役立ちます。ただし、ログやメモに secret、stream key、Webhook URL を貼らないでください。
