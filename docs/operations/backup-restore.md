# バックアップと復元

AutoStream の設定や運用データを失わないための基本手順です。実際の secret はバックアップ先でも秘密情報として扱ってください。

## バックアップするもの

- database のバックアップ
- env ファイルや systemd 設定
- compose ファイル
- 録画保存先の必要なファイル
- reverse proxy の設定
- 復元手順のメモ

## バックアップ手順

1. 配信中でないことを確認します。
2. database の dump を作成します。
3. env ファイルと起動設定を安全な場所へ保存します。
4. 録画ファイルを別ディスクや外部保存先へ退避します。
5. バックアップに実際の token が含まれる場合は、アクセス権を制限します。
6. 作成日時と対象サービスをメモします。

## 復元手順

1. 新しいサーバーに必要なパッケージを入れます。
2. artifact、compose、または systemd 設定を配置します。
3. database を復元します。
4. env ファイルを配置します。
5. サービスを起動します。
6. Control Panel にログインし、配信先、保存先、通知先を確認します。
7. 短いテスト配信を行います。

本番復元の前に、別環境で復元テストを一度行うのがおすすめです。
