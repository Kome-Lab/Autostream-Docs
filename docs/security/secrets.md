# 秘密情報の扱い

token、配信キー、Webhook URL、認証ファイルなどの扱い方です。秘密情報は動作に必要ですが、公開リポジトリや共有資料には載せません。

## 置く場所

- サーバーの env ファイル
- secret store
- Control Panel の設定画面
- 権限を制限した認証ファイル

## 置かない場所

- GitHub の公開リポジトリ
- README や公開ドキュメント
- issue やチャット
- スクリーンショット
- 誰でも見られるログ

## 入れ替えるタイミング

- 誤って公開した可能性がある
- 担当者が変わった
- 不要な連携先を削除した
- 定期的なメンテナンスを行う

入れ替えたあとは、古い token や Webhook URL を provider 側で無効化してください。
