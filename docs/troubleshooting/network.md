# ネットワークの確認

Control Panel、Worker、Encoder Recorder、配信先、保存先につながらない場合の確認手順です。

## 確認する順番

1. サーバーが起動しているか確認します。
2. Control Panel の URL をブラウザで開きます。
3. reverse proxy が動いているか確認します。
4. firewall で必要なポートが閉じていないか確認します。
5. サービス間の内部 URL が正しいか確認します。
6. 配信先や保存先にサーバーから接続できるか確認します。

## よくある原因

- 公開 URL と内部 URL を取り違えている
- HTTPS の証明書が正しくない
- firewall で必要な通信を止めている
- DNS が古い向き先を見ている
- provider 側の stream key や認証が無効になっている

## ログを見るとき

接続エラーのログには URL や token の一部が出ることがあります。共有する前に secret が含まれていないか確認してください。
